🤔 Next.js에서 Gatsby로 블로그 이사 중간 리뷰

해당 블로그 제작 중간 리뷰

기존 블로그

❓ Next.js 선택 이유

  • SPA의 장점과 SSR의 장점을 모두 가짐
  • 리액트를 사용해봤다면 쉽게 사용 가능
  • 별도 라이브러리 없이 간편한 라우팅을 지원
  • Typescript 지원

🤔 사용하면서 느낀점

정보 부족

대중적인 조합이 아니라 오류에 대한 정보를 찾기가 어려웠습니다.

편의성

작성한 포스트의 배포 과정이 너무 복잡했고 잔 오류가 너무 많았습니다.

분명 Gh-pages와 Next.js만 사용할 때는 괜찮았는데 Typescript를 이용해 작성하면 정적 사이트로 정상 빌드되지만 Github에 정상적으로 Push 되지 않는 경우가 많았습니다.

(한두 번이면 이해하겠는데 생각보다 자주 먹통이었음)

이사를 결심한 이유

처음에는 새로 만들 엄두가 안 나서 잔 오류들을 수정해가면서 사용했는데 Dependencies가 조금만 바뀌어도 다시 오류가 생기고 포스팅 시간보다 이런 문제들을 해결하는데 더 많은 시간이 들었습니다.


그래도 완전히 새로 만들기보다는 포스팅 때마다 문제가 생겼던 Gh-pages를 포기하고 다른 방법으로 배포하려 했습니다.


가장 처음 생각난 건 AWS를 이용한 호스팅이었지만 한번 만들어두면 계속 쓸 거라는 생각에 기간 한정 무료(유료) 서비스는 이용하기 싫었고 평소 토이 프로젝트에 자주 사용했던 Heroku로 배포해봤는데 서버가 Sleep상태면 서버를 다시 깨우는데 시간이 너무 오래 걸려서 결국 무료로 웹을 호스팅 할 수 있는 건 정적 웹호스팅 서비스가 가장 적합하다는 생각이었습니다.


그리고 나중에 알게 된 거지만 SPA + SSR의 장점 때문에 Next.js를 사용하려는데 Gh-pages를 이용한 정적 웹호스팅에서는 해당 장점들은 크게 의미 없어진다고 합니다.

새로운 블로그

✅ 새로운 블로그 고려사항

  • Typescript 지원

    • 한번 만들어두면 계속 쓸 텐데 유지 보수하기 확실히 Javascript보단 (개인적으로) 편하다고 느꼈고 공식적으로 Typescript를 지원하는 도구를 사용하고 싶었습니다.

  • Gh-pages를 이용한 배포

    • Github page에 호스팅 할 수 있는 여러 가지 도구들이 있지만 좀 더 안정적으로 관리할 수 있도록 정적 웹으로의 변환을 지원하는 도구를 사용하고 싶었습니다.

📜 후보

  • Jekyll

    • 간편하게 블로그 작성 가능
    • Ruby가 익숙하지 않아 정해진 템플릿에서 큰 수정이 어려움

  • Gatsby

    • 리액트 기반, 라이브러리 대부분 익숙함
    • GraphQL을 통해 간편하게 Data 관리 가능
    • Typescript 설정이 리액트와 별 차이가 없음


위 조건을 만족하는 도구는 크게 두 가지로 Jekyll, Gatsby가 있었고 둘 다 사용해보면서 새삼 개발 도구 선택이 얼마나 중요한지 느꼈습니다.

그동안 많은 시간을 투자해야 사용할 수 있던 기능들이 기본적으로 구현돼있는 라이브러리들이 많았고 남들이 많이 쓰는 데는 이유가 있었습니다.

그 결과 위와 같은 이유로 Gatsby가 저한테는 더 맞는 것 같았습니다.

(관련 커뮤니티는 Jekyll가 더 활성화되어있는 듯)

Gatsby 블로그 템플릿

일단 Gatsby에 공유되어있는 템플릿 중에 정말 괜찮은 디자인이 많아서 템플릿 중 하나를 수정해서 사용하려 했지만 블로그 제작을 목적으로 하는 템플릿은 마음에 드는게 없어 다른 용도의 템플릿 중 괜찮은 것을 찾아 몇 가지 기능을 추가해서 사용하려 했습니다.

하지만 기능을 추가해 가면서 다음과 같은 문제점이 있어서 그냥 템플릿 없이 기본 Gatsby 설정으로 새 블로그를 만들게 되었습니다.

  • 사용하지 않는 잡다한 기능들이 많음
  • 최신 CSS로 구현 가능한 기능들이 JS로 구현되어있음
  • 원래 탬플릿의 스타일과 내가 새로 작성한 스타일이 충돌하는 경우가 있음
  • 위와 같은 문제점을 수정하는데 너무 많은 시간을 쓰게 됨

마지막으로

앞으로도 해당 블로그를 관리하고 필요한 기능들도 마저 구현해야겠지만 일단 도구 선택은 만족합니다.

이미 블로그에 포스팅도 가능하고 어느 정도 사용이 가능해서 당장 이번 글부터 포스팅해나가겠지만 추후 모든 기능이 완성되고 SEO까지 마친다면 블로그 제작 리뷰를 다시 한번 작성할 예정입니다.


비록 저는 템플릿을 사용하지 않았지만 Gatsby를 통해 자신만의 블로그를 만들어 보고 싶으신 분들은

Gatsby Starter Library에서 여러가지 템플릿을 확인 할 수 있습니다.
